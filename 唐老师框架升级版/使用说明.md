# 唐老师框架迭代版使用说明

## 1. 框架概述

本框架基于唐老师《程序小框架重制版》中的框架进行了一些优化

### 1.1 主要特性

- **完整的资源管理**：支持 AB 包资源加载、编辑器资源加载等多种资源管理方式
- **UI 框架**：提供了基于面板的 UI 管理系统，支持动画和扩展
- **事件中心**：实现了基于事件的通信机制，降低模块间耦合
- **对象池系统**：支持 GameObject 和引用类型的对象池，提高性能
- **有限状态机**：提供了通用的状态机实现，支持各种游戏逻辑状态管理
- **Excel 导表工具**：支持将 Excel 表格数据导出为二进制或 JSON 格式
- **计时器系统**：提供了高效的计时器管理功能
- **音效管理**：统一管理游戏中的音效和音乐
- **日志系统**：提供了可控制的日志输出，支持开发环境和发布环境的不同配置
- **输入管理**：统一处理游戏输入
- **场景加载**：提供了场景加载的管理功能
- **摄像机管理**：支持多摄像机管理和切换
- **代码测速工具**：帮助开发者分析代码性能

## 3. 核心模块使用说明

### 3.1 单例模式

框架提供了多种单例实现，方便创建全局唯一的管理器类：

- **BaseManager** **`<T>`**：基于静态字段的单例实现，适用于大多数管理器类
- **SingletonMono** **`<T>`**：继承自 MonoBehaviour 的单例实现，适用于需要挂载到游戏对象的类
- **SingletonAutoMono** **`<T>`**：自动创建游戏对象并挂载的单例实现

#### 使用示例

```csharp
public class MyManager : BaseManager<MyManager>
{
    // 成员变量和方法
}

// 使用方式
MyManager.Instance.DoSomething();
```

### 3.2 资源加载系统

框架提供了多种资源加载方式，适应不同场景的需求：

#### 3.2.1 AB 包资源加载

**ABMgr**：管理 AB 包的加载和卸载

```csharp
// 同步加载资源
GameObject obj = ABMgr.Instance.LoadRes<GameObject>("ab包名", "资源名");

// 异步加载资源
ABMgr.Instance.LoadResAsync<GameObject>("ab包名", "资源名", (obj) => {
    // 资源加载完成后的回调
});
```

**ABResourceLoader**：基于配置的 AB 包资源加载器

```csharp
// 设置配置文件路径
ABResourceLoader.Instance.SetConfigFilePath("AB包配置文件/ABResourceConfig.json");

// 加载资源
GameObject obj = ABResourceLoader.Instance.LoadResource<GameObject>("ab包名", "资源名");
```

#### 3.2.2 编辑器资源加载

**EditorResMgr**：在编辑器模式下直接加载资源

```csharp
// 加载音频资源
AudioClip clip = EditorResMgr.Instance.LoadAudioClip("music", "背景音乐");

// 加载音效资源
AudioClip sound = EditorResMgr.Instance.LoadSoundClip("sound", "按钮音效");
```

#### 3.2.3 常规资源加载

**ResMgr**：管理 Resources 文件夹下的资源加载

```csharp
// 同步加载资源
GameObject obj = ResMgr.Instance.LoadRes<GameObject>("Prefabs/Player");

// 异步加载资源
ResMgr.Instance.LoadResAsync<GameObject>("Prefabs/Player", (obj) => {
    // 资源加载完成后的回调
});
```

### 3.3 UI 框架

**UIMgr**：管理 UI 面板的显示、隐藏和切换

```csharp
// 显示面板
UIMgr.Instance.ShowPanel<MainPanel>();

// 隐藏面板
UIMgr.Instance.HidePanel<MainPanel>();

// 获取面板实例
MainPanel panel = UIMgr.Instance.GetPanel<MainPanel>();
```

**BasePanel**：所有 UI 面板的基类

```csharp
public class MainPanel : BasePanel
{
    // 面板初始化
    public override void OnInit()
    {
        // 初始化面板
    }
  
    // 面板显示
    public override void ShowMe(object args)
    {
        // 面板显示时的逻辑
    }
  
    // 面板隐藏
    public override void HideMe()
    {
        // 面板隐藏时的逻辑
    }
}
```

### 3.4 事件中心

**EventCenter**：实现模块间的事件通信

```csharp
// 注册事件监听器
EventCenter.Instance.AddEventListener<E_EventType>(E_EventType.PlayerDead, OnPlayerDead);

// 触发事件
EventCenter.Instance.EventTrigger<E_EventType>(E_EventType.PlayerDead);

// 移除事件监听器
EventCenter.Instance.RemoveEventListener<E_EventType>(E_EventType.PlayerDead, OnPlayerDead);

// 事件处理方法
private void OnPlayerDead()
{
    // 处理玩家死亡事件
}
```

### 3.5 对象池系统

#### 3.5.1 GameObject 对象池

**GOPoolMgr**：管理游戏对象的对象池

```csharp
// 从对象池获取对象
GameObject obj = GOPoolMgr.Instance.GetObj("Prefabs/Player");

// 回收对象到对象池
GOPoolMgr.Instance.PushObj(obj);
```

#### 3.5.2 引用对象池

**ReferencePoolMgr**：管理引用类型的对象池

```csharp
// 从对象池获取对象
MyClass obj = ReferencePoolMgr.Instance.GetObj<MyClass>();

// 回收对象到对象池
ReferencePoolMgr.Instance.PushObj(obj);
```

### 3.6 有限状态机

**StateMachine**：通用的状态机实现

```csharp
// 创建状态机
StateMachine<PlayerStateType, Player> stateMachine = new StateMachine<PlayerStateType, Player>(player);

// 添加状态
stateMachine.AddState(PlayerStateType.Idle, new PlayerIdleState());
stateMachine.AddState(PlayerStateType.Run, new PlayerRunState());

// 设置初始状态
stateMachine.SetCurrentState(PlayerStateType.Idle);

// 切换状态
stateMachine.ChangeState(PlayerStateType.Run);
```

### 3.7 计时器系统

**TimerMgr**：管理游戏中的计时器

```csharp
// 创建计时器
int timerId = TimerMgr.Instance.CreateTimer(false, 3000, () => {
    // 3秒后执行的回调
}, 1000, () => {
    // 每1秒执行一次的回调
});

// 停止计时器
TimerMgr.Instance.StopTimer(timerId);

// 重置计时器
TimerMgr.Instance.ResetTimer(timerId);
```

### 3.8 音效管理

**MusicMgr**：管理游戏中的音乐和音效

```csharp
// 播放背景音乐
MusicMgr.Instance.PlayMusic("背景音乐", 0.5f, true);

// 播放音效
MusicMgr.Instance.PlaySound("按钮点击", 1f);

// 停止背景音乐
MusicMgr.Instance.StopMusic();

// 暂停音效
MusicMgr.Instance.PauseSound();
```

### 3.9 日志系统

**LogSystem**：提供可控制的日志输出


### 3.10 场景加载

**SceneMgr**：管理场景的加载和切换

```csharp
// 同步加载场景
SceneMgr.Instance.LoadScene("GameScene");

// 异步加载场景
SceneMgr.Instance.LoadSceneAsyn("GameScene", (progress) => {
    // 加载进度回调
```

### 3.11 输入管理

**InputMgr**：统一处理游戏输入

```csharp

// 注册按键抬起事件
```

## 4. 编辑器工具

### 4.1 Excel 导表工具

**ExcelTool**：将 Excel 表格数据导出为游戏可用的格式

1. 在 `Editor/Excel/ExcelTool.cs` 中配置 Excel 文件路径和输出路径
2. 在 Unity 编辑器中执行 `Tools/Excel工具/导出二进制数据` 或 `Tools/Excel工具/导出JSON数据` 菜单命令
3. 导出的数据会生成到指定的输出路径

### 4.2 AB 包工具

**ABTools**：管理 AB 包的创建和上传

1. **CreateABCompare**：创建 AB 包对比文件
2. **MoveABToSA**：将 AB 包移动到 StreamingAssets 目录
3. **UploadAB**：上传 AB 包到服务器

### 4.3 路径配置工具

**PathConfigWindow**：可视化配置框架中的各种路径

1. 在 Unity 编辑器中执行 `Tools/路径配置` 菜单命令
2. 在弹出的窗口中设置各种路径
3. 点击 "保存配置" 按钮保存设置

### 4.4 AB 资源编辑器窗口

**ABResourceEditorWindow**：可视化管理 AB 包资源

1. 在 Unity 编辑器中执行 `Tools/AB资源管理` 菜单命令
2. 在弹出的窗口中查看和管理 AB 包资源
3. 点击 "保存配置" 按钮生成配置文件

## 5. 配置文件

### 5.1 PathConfig.json

存储框架中各种工具的路径配置：

```json
{
  "excelPath": "Excel表数据路径",
  "dataClassPath": "数据类生成路径",
  "dataContainerPath": "数据容器生成路径",
  "binaryOutputPath": "二进制数据输出路径",
  "jsonOutputPath": "JSON数据输出路径",
  "editorResRootPath": "编辑器资源根路径",
  "abResourceConfigPath": "AB资源配置文件路径"
}
```

### 5.2 ABResourceConfig.json

存储 AB 包资源的配置信息：

```json
{
  "resources": [
    {
      "ResourceName": "资源名称",
      "ResourcePath": "资源路径",
      "ABPackageName": "AB包名称"
    }
  ]
}
```
